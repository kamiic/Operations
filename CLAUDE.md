# Operations リポジトリ — Claude Code 指示書

このリポジトリは、上市星也（Ema）専用の汎用業務エージェント運用基盤。
目的は「一度起きたミスを二度と発生させない」こと。

## 起動時に必ず行うこと

1. `rules/MASTER_RULES.md` を読む（全業務共通ルール）
2. 該当する業務がある場合、`rules/[業務名].md` も読む
3. 上記ルールに違反しない範囲で作業・回答を行う

## 絶対に守ること

- マスタードキュメント／前提情報より先に成果物・回答を作らない
- ユーザーへのヒアリングは一度に1問のみ。複数の質問を同時に提示しない
- 自分の判断で複数の論点をまとめて確認しようとせず、まず1つだけ聞く
- run_check.py 等のチェックツールがある業務では、チェックでOKになるまで完了報告しない

## ミスが発生した場合の対応フロー

1. ユーザーから「これはミスだ」という指摘を受けたら、まず謝罪より先に原因を特定する
2. そのミスが `rules/MASTER_RULES.md` または該当業務ルールに既に記載されているか確認する
3. 記載がなければ、新しいルールとして追記する（このリポジトリの `tools/auto_fix_add_detection` 相当の人力版）
4. 同じパターンのミスを二度と起こさないよう、ルール文を具体的に書く（抽象的な反省ではなく、行動レベルの禁止事項にする）

## ルール追記のフォーマット

```
### [日付] [ミスの概要]
- 何が起きたか：
- なぜダメか：
- 次回からどうするか（行動レベル）：
```
